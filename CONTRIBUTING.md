# Contributing to SGEN

Thank you for your interest in contributing to SGEN! This project is open for community-driven development and improvements.

## What We Welcome

- New features and feature requests
- UI/UX improvements
- Performance optimizations
- Code refactoring and cleanups
- Documentation improvements
- Translations

## What the Maintainer Handles

The project owner will primarily manage the overall direction of the project. **Bug fixes and issue triage are not the main focus** of the maintainer, so the community is encouraged to help investigate and resolve bugs through pull requests.

## Getting Started

1. Fork the repository
2. Create a feature branch: `git checkout -b feature/my-feature`
3. Set up your local environment using the instructions in [README.md](./README.md)
4. Make your changes
5. Open a pull request against the `main` branch

## Development Guidelines

- Follow the existing code style and project structure
- Keep changes focused and minimal
- Add or update tests when applicable
- Ensure the application builds and starts without errors
- Write clear commit messages

## Third-Party Respect

SGEN integrates with **Intervals.icu** for training data. We have great respect for the creators of Intervals.icu and their platform. Contributors must:

- Not abuse the Intervals.icu API
- Follow Intervals.icu terms of service and API guidelines
- Avoid actions that could harm or overload the Intervals.icu service
- Acknowledge that Intervals.icu is a separate product and SGEN is not affiliated with or endorsed by Intervals.icu

## Pull Requests

- Describe what your change does and why
- Reference any related issues
- Keep pull requests focused on a single concern
- Be respectful and constructive in code reviews

## Questions?

For questions, ideas, or discussions, please open a GitHub Discussion or Issue.

Thank you for helping make SGEN better!
